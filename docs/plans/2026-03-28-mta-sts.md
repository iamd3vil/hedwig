# MTA-STS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development (recommended) or executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement sending-side MTA-STS (RFC 8461) so Hedwig enforces recipient domain TLS policies during outbound delivery.

**Architecture:** New `mta_sts` module in `smtp-server/src/` with four files: policy types + parsing, DNS/HTTPS fetcher, moka-backed cache, and a background refresh task. The `Worker` gains an `Arc<MtaStsResolver>` and checks MTA-STS policy before each MX delivery attempt. On `enforce` mode, MXes that fail validation are skipped; on `testing`, failures are logged but delivery proceeds.

**Tech Stack:** Rust, tokio, hickory-resolver (existing), reqwest (new — HTTPS fetch), moka (existing — caching), lettre (existing — SMTP delivery)

**Spec:** `docs/superpowers/specs/2026-03-28-mta-sts-design.md`

### Task 1: Policy Types & MX Pattern Matching

Pure data types and parsing logic with no I/O. This is the foundation everything else builds on.

**Files:**
- Create: `smtp-server/src/mta_sts/policy.rs`
- Create: `smtp-server/src/mta_sts/mod.rs`

**Context:** MTA-STS policies are fetched from HTTPS endpoints and contain fields like `version: STSv1`, `mode: enforce`, `mx: *.example.com`, `max_age: 604800`. The policy text format is CRLF-separated `key: value` pairs. MX pattern matching follows RFC 6125: wildcards like `*.example.com` match exactly one left-most label (e.g. `mail.example.com` matches, but `example.com` and `a.b.example.com` do not). Patterns and hostnames are compared case-insensitively. MX hostnames from DNS often have a trailing dot (e.g. `mail.example.com.`) which must be stripped before matching.

use std::fmt;
use std::time::Instant;

/// MTA-STS policy mode (RFC 8461 §3.2).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PolicyMode {
    /// Sending MTAs MUST NOT deliver to hosts that fail validation.
    Enforce,
    /// Failures are reported but delivery proceeds.
    Testing,
    /// Domain wishes to disable MTA-STS.
    None,
}

impl fmt::Display for PolicyMode {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            PolicyMode::Enforce => write!(f, "enforce"),
            PolicyMode::Testing => write!(f, "testing"),
            PolicyMode::None => write!(f, "none"),
        }
    }
}

/// A parsed MTA-STS policy (RFC 8461 §3.2).
#[derive(Debug, Clone)]
pub struct MtaStsPolicy {
    pub mode: PolicyMode,
    pub mx_patterns: Vec<String>,
    pub max_age: u64,
}

/// A cached policy entry with metadata for freshness tracking.
#[derive(Debug, Clone)]
pub struct CachedPolicy {
    pub policy: MtaStsPolicy,
    pub txt_id: String,
    pub fetched_at: Instant,
}

/// Parsed `_mta-sts` TXT record fields.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct MtaStsTxtRecord {
    pub id: String,
}

/// Errors that can occur during MTA-STS policy parsing.
#[derive(Debug, thiserror::Error)]
pub enum PolicyParseError {
    #[error("missing required field: {0}")]
    MissingField(&'static str),
    #[error("invalid version: expected STSv1, got {0}")]
    InvalidVersion(String),
    #[error("invalid mode: {0}")]
    InvalidMode(String),
    #[error("invalid max_age: {0}")]
    InvalidMaxAge(String),
}

/// Parse raw MTA-STS TXT record text into structured fields.
///
/// Expected format: `v=STSv1; id=20160831085700Z;`
/// Multiple key=value pairs separated by semicolons.
pub fn parse_txt_record(txt: &str) -> Result<MtaStsTxtRecord, PolicyParseError> {
    let mut version_found = false;
    let mut id: Option<String> = None;

    for part in txt.split(';') {
        let part = part.trim();
        if part.is_empty() {
            continue;
        }
        if let Some((key, value)) = part.split_once('=') {
            let key = key.trim();
            let value = value.trim();
            match key {
                "v" => {
                    if value != "STSv1" {
                        return Err(PolicyParseError::InvalidVersion(value.to_string()));
                    }
                    version_found = true;
                }
                "id" => {
                    id = Some(value.to_string());
                }
                _ => {
                    // Unknown fields are ignored per RFC 8461 §3.2
                }
            }
        }
    }

    if !version_found {
        return Err(PolicyParseError::MissingField("v"));
    }
    let id = id.ok_or(PolicyParseError::MissingField("id"))?;

    Ok(MtaStsTxtRecord { id })
}

/// Parse an MTA-STS HTTPS policy body into structured fields.
///
/// Expected format: CRLF-separated `key: value` pairs.
/// Example:
/// ```text
/// version: STSv1
/// mode: enforce
/// mx: mail.example.com
/// mx: *.example.net
/// max_age: 604800
/// ```
pub fn parse_policy(body: &str) -> Result<MtaStsPolicy, PolicyParseError> {
    let mut version_found = false;
    let mut mode: Option<PolicyMode> = None;
    let mut mx_patterns: Vec<String> = Vec::new();
    let mut max_age: Option<u64> = None;

    for line in body.lines() {
        let line = line.trim_end_matches('\r').trim();
        if line.is_empty() {
            continue;
        }
        if let Some((key, value)) = line.split_once(':') {
            let key = key.trim();
            let value = value.trim();
            match key {
                "version" => {
                    if value != "STSv1" {
                        return Err(PolicyParseError::InvalidVersion(value.to_string()));
                    }
                    version_found = true;
                }
                "mode" => {
                    if mode.is_none() {
                        mode = Some(match value {
                            "enforce" => PolicyMode::Enforce,
                            "testing" => PolicyMode::Testing,
                            "none" => PolicyMode::None,
                            other => return Err(PolicyParseError::InvalidMode(other.to_string())),
                        });
                    }
                    // Duplicate non-mx fields: keep first per RFC §3.2
                }
                "mx" => {
                    mx_patterns.push(value.to_lowercase());
                }
                "max_age" => {
                    if max_age.is_none() {
                        max_age = Some(
                            value
                                .parse::<u64>()
                                .map_err(|_| PolicyParseError::InvalidMaxAge(value.to_string()))?,
                        );
                    }
                }
                _ => {
                    // Unknown fields ignored per RFC
                }
            }
        }
    }

    if !version_found {
        return Err(PolicyParseError::MissingField("version"));
    }
    let mode = mode.ok_or(PolicyParseError::MissingField("mode"))?;
    let max_age = max_age.ok_or(PolicyParseError::MissingField("max_age"))?;

    // mx is required at least once unless mode is none
    if mode != PolicyMode::None && mx_patterns.is_empty() {
        return Err(PolicyParseError::MissingField("mx"));
    }

    Ok(MtaStsPolicy {
        mode,
        mx_patterns,
        max_age,
    })
}

/// Check whether an MX hostname matches an MTA-STS policy `mx:` pattern.
///
/// Matching rules (RFC 8461 §4.1, RFC 6125):
/// - Exact match is case-insensitive.
/// - Wildcard `*.example.com` matches exactly one left-most label
///   (e.g. `mail.example.com` matches, `example.com` does not,
///   `a.b.example.com` does not).
///
/// The `mx_host` may have a trailing dot from DNS (e.g. `mail.example.com.`)
/// which is stripped before comparison.
pub fn mx_matches_pattern(mx_host: &str, pattern: &str) -> bool {
    let mx_host = mx_host.trim_end_matches('.').to_lowercase();
    let pattern = pattern.trim_end_matches('.').to_lowercase();

    if pattern.starts_with("*.") {
        let suffix = &pattern[1..]; // e.g. ".example.com"
        match mx_host.split_once('.') {
            Some((_, rest)) => format!(".{}", rest) == suffix,
            None => false,
        }
    } else {
        mx_host == pattern
    }
}

/// Check whether an MX hostname matches any pattern in the policy.
pub fn mx_matches_policy(mx_host: &str, policy: &MtaStsPolicy) -> bool {
    policy
        .mx_patterns
        .iter()
        .any(|pattern| mx_matches_pattern(mx_host, pattern))
}

**Context:** The fetcher has two responsibilities: (1) look up `_mta-sts.<domain>` TXT record via DNS to get the policy `id`, and (2) fetch the policy body from `https://mta-sts.<domain>/.well-known/mta-sts.txt` via HTTPS. The fetcher uses `hickory_resolver::TokioAsyncResolver` (same type the `Worker` already creates from system config) for DNS and `reqwest::Client` for HTTPS.

```rust
use hickory_resolver::{
    name_server::{GenericConnector, TokioRuntimeProvider},
    AsyncResolver,
};
use miette::{Context, IntoDiagnostic, Result};
use tracing::{debug, warn};

use super::policy::{self, MtaStsPolicy, MtaStsTxtRecord};

/// Maximum policy body size (64 KB, per RFC 8461 §3.3).
const MAX_POLICY_BODY_SIZE: usize = 64 * 1024;

/// HTTPS request timeout (60 seconds, per RFC 8461 §3.3).
const FETCH_TIMEOUT: std::time::Duration = std::time::Duration::from_secs(60);

/// Handles DNS TXT lookups and HTTPS policy fetching for MTA-STS.
pub struct MtaStsFetcher {
    resolver: AsyncResolver<GenericConnector<TokioRuntimeProvider>>,
    http_client: reqwest::Client,
}

impl MtaStsFetcher {
    /// Create a new fetcher using the given DNS resolver.
    ///
    /// The `reqwest::Client` is configured per RFC 8461 requirements:
    /// no redirect following, 60-second timeout.
    pub fn new(resolver: AsyncResolver<GenericConnector<TokioRuntimeProvider>>) -> Result<Self> {
        let http_client = reqwest::Client::builder()
            .redirect(reqwest::redirect::Policy::none())
            .timeout(FETCH_TIMEOUT)
            .build()
            .into_diagnostic()
            .wrap_err("building HTTP client for MTA-STS")?;

        Ok(Self {
            resolver,
            http_client,
        })
    }

    /// Look up the `_mta-sts.<domain>` DNS TXT record.
    ///
    /// Returns `Ok(None)` if no valid MTA-STS TXT record exists.
    /// Returns `Err` only on DNS transport-level errors (not NXDOMAIN).
    pub async fn lookup_txt(&self, domain: &str) -> Result<Option<MtaStsTxtRecord>> {
        let name = format!("_mta-sts.{}", domain);
        debug!(%domain, dns_name = %name, "looking up MTA-STS TXT record");

        let txt_lookup = match self.resolver.txt_lookup(&name).await {
            Ok(lookup) => lookup,
            Err(err) => {
                // NXDOMAIN or NODATA means no MTA-STS — not an error.
                use hickory_resolver::error::ResolveErrorKind;
                match err.kind() {
                    ResolveErrorKind::NoRecordsFound { .. } => {
                        debug!(%domain, "no MTA-STS TXT record found");
                        return Ok(None);
                    }
                    _ => {
                        return Err(err)
                            .into_diagnostic()
                            .wrap_err("DNS TXT lookup for MTA-STS");
                    }
                }
            }
        };

        // Filter for records starting with "v=STSv1;".
        // Per RFC 8461 §3.1: discard records not starting with "v=STSv1;",
        // and if the number of remaining records is not exactly one, skip.
        let valid_records: Vec<String> = txt_lookup
            .iter()
            .map(|txt| {
                txt.iter()
                    .map(|data| String::from_utf8_lossy(data).to_string())
                    .collect::<String>()
            })
            .filter(|s| s.starts_with("v=STSv1;") || s.starts_with("v=STSv1 ;"))
            .collect();

        if valid_records.len() != 1 {
            if valid_records.len() > 1 {
                warn!(%domain, count = valid_records.len(), "multiple MTA-STS TXT records found, skipping");
            }
            return Ok(None);
        }

        match policy::parse_txt_record(&valid_records[0]) {
            Ok(record) => {
                debug!(%domain, id = %record.id, "found MTA-STS TXT record");
                Ok(Some(record))
            }
            Err(err) => {
                warn!(%domain, error = %err, "invalid MTA-STS TXT record");
                Ok(None)
            }
        }
    }

    /// Fetch the MTA-STS policy from `https://mta-sts.<domain>/.well-known/mta-sts.txt`.
    ///
    /// Returns `Ok(None)` if the policy could not be fetched (non-200, timeout, etc.).
    /// Per RFC 8461 §3.3: only HTTP 200 is valid, redirects MUST NOT be followed.
    pub async fn fetch_policy(&self, domain: &str) -> Result<Option<MtaStsPolicy>> {
        let url = format!("https://mta-sts.{}/.well-known/mta-sts.txt", domain);
        debug!(%domain, %url, "fetching MTA-STS policy");

        let response = match self.http_client.get(&url).send().await {
            Ok(resp) => resp,
            Err(err) => {
                warn!(%domain, error = %err, "failed to fetch MTA-STS policy");
                return Ok(None);
            }
        };

        if response.status() != reqwest::StatusCode::OK {
            warn!(%domain, status = %response.status(), "MTA-STS policy fetch returned non-200");
            return Ok(None);
        }

        // Validate Content-Type is text/plain (SHOULD per RFC 8461 §3.2).
        if let Some(content_type) = response.headers().get(reqwest::header::CONTENT_TYPE) {
            if let Ok(ct) = content_type.to_str() {
                if !ct.starts_with("text/plain") {
                    warn!(%domain, content_type = %ct, "MTA-STS policy has unexpected Content-Type");
                    // Continue anyway — RFC says SHOULD, not MUST.
                }
            }
        }

        // Read body with size limit.
        let body = match response.text().await {
            Ok(text) => {
                if text.len() > MAX_POLICY_BODY_SIZE {
                    warn!(%domain, size = text.len(), "MTA-STS policy body exceeds 64KB limit");
                    return Ok(None);
                }
                text
            }
            Err(err) => {
                warn!(%domain, error = %err, "failed to read MTA-STS policy body");
                return Ok(None);
            }
        };

        match policy::parse_policy(&body) {
            Ok(p) => {
                debug!(%domain, mode = %p.mode, max_age = p.max_age, mx_count = p.mx_patterns.len(), "parsed MTA-STS policy");
                Ok(Some(p))
            }
            Err(err) => {
                warn!(%domain, error = %err, "failed to parse MTA-STS policy");
                Ok(None)
            }
        }
    }
}
```

### Task 3: MTA-STS Policy Cache (MtaStsResolver)

The main public API that combines fetching + caching. Workers call `resolver.get_policy(domain)`.

**Files:**
- Create: `smtp-server/src/mta_sts/cache.rs`
- Modify: `smtp-server/src/mta_sts/mod.rs`

**Context:** The cache is a moka in-memory cache keyed by domain, with TTL from the policy's `max_age`. The `MtaStsResolver` provides `get_policy(domain)` which: (1) checks the cache, (2) looks up the DNS TXT record, (3) if the TXT `id` matches the cached entry, returns the cached policy without an HTTPS fetch, (4) otherwise fetches the HTTPS policy and caches it. If fetching fails but a valid cached policy exists, the cached policy is returned (RFC requirement). The resolver also tracks fetch-failure cooldowns (5 minutes per domain) to avoid overwhelming failing endpoints.

Create `smtp-server/src/mta_sts/cache.rs`:

```rust
    ///
    /// Returns `Some(policy)` if a valid policy exists, `None` if the domain
    /// doesn't participate in MTA-STS or the policy couldn't be fetched.
    ///
    /// Implements the full RFC 8461 cache-check → DNS → HTTPS flow:
    /// 1. Look up DNS TXT record for `_mta-sts.<domain>`
    /// 2. If cached policy exists with matching TXT id → return cached
    /// 3. If TXT id differs or no cache → fetch HTTPS policy
    /// 4. On fetch failure, fall back to cached policy if available
    pub async fn get_policy(&self, domain: &str) -> Option<MtaStsPolicy> {

        // Step 1: DNS TXT lookup.
        let txt_record = match self.fetcher.lookup_txt(&domain).await {
            Ok(Some(record)) => record,
            Ok(None) => {
                // No MTA-STS TXT record. If we have a cached non-expired policy,
                // keep using it (RFC 8461 §3.3). Otherwise, no policy.
                return self.get_cached_policy(&domain);
            }
            Err(err) => {
                warn!(%domain, error = %err, "DNS lookup error for MTA-STS, using cached policy if available");
                return self.get_cached_policy(&domain);
            }
        };

        // Step 2: Check cache. If txt_id matches, return cached policy.
        if let Some(cached) = self.cache.get(&domain).await {
            if cached.txt_id == txt_record.id {
                debug!(%domain, id = %txt_record.id, "MTA-STS policy cache hit (id matches)");
                return Some(cached.policy.clone());
            }
            debug!(%domain, cached_id = %cached.txt_id, new_id = %txt_record.id, "MTA-STS policy id changed, re-fetching");
        }

        // Step 3: Check fetch failure cooldown.
        if self.failure_cooldowns.get(&domain).await.is_some() {
            debug!(%domain, "MTA-STS fetch in cooldown, using cached policy if available");
            return self.get_cached_policy(&domain);

        match self.fetcher.fetch_policy(&domain).await {
            Ok(Some(policy)) => {

                    policy: policy.clone(),
                    txt_id: txt_record.id,
                    fetched_at: Instant::now(),

                self.cache.insert(domain.clone(), cached).await;

                info!(%domain, mode = %policy.mode, max_age = policy.max_age, "cached MTA-STS policy");
                Some(policy)

                // Fetch failed (non-200, timeout, parse error). Record cooldown.
                self.failure_cooldowns.insert(domain.clone(), Instant::now()).await;
                // Fall back to cached policy per RFC 8461 §3.3.
                self.get_cached_policy(&domain)

            Err(err) => {
                warn!(%domain, error = %err, "error fetching MTA-STS policy");
                self.failure_cooldowns.insert(domain.clone(), Instant::now()).await;
                self.get_cached_policy(&domain)
```

### Task 4: Background Policy Refresher

A tokio task that proactively refreshes cached policies before they expire.

**Files:**
- Create: `smtp-server/src/mta_sts/refresher.rs`
- Modify: `smtp-server/src/mta_sts/mod.rs`

**Context:** Per RFC 8461 §10.2, senders SHOULD proactively refresh cached policies before they expire, and SHOULD alert administrators on persistent refresh failures. This task runs daily, iterates all cached entries, checks if the TXT record `id` has changed, and re-fetches if needed. It extends the freshness of unchanged policies. It skips policies with mode `none` (per RFC guidance for clean opt-out in §8.3). The refresher uses the shared `MtaStsResolver` (specifically its cache and fetcher) and listens on the global `CancellationToken` for graceful shutdown.

```rust
use std::sync::Arc;
use std::time::{Duration, Instant};

use tokio_util::sync::CancellationToken;
use tracing::{debug, info, warn};

use super::cache::MtaStsResolver;
use super::policy::{CachedPolicy, PolicyMode};

/// Refresh interval: once per day (per RFC 8461 §10.2 suggestion).
const REFRESH_INTERVAL: Duration = Duration::from_secs(24 * 60 * 60);

/// Run the background policy refresh loop.
///
/// Iterates all cached MTA-STS policies, checks for updates, and re-fetches
/// when the TXT record `id` has changed. Extends freshness of unchanged policies.
///
/// Runs until the `shutdown` token is cancelled.
pub async fn run_refresh_loop(resolver: Arc<MtaStsResolver>, shutdown: CancellationToken) {
    info!("MTA-STS background refresher started (interval: 24h)");

    let mut interval = tokio::time::interval(REFRESH_INTERVAL);
    interval.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Delay);

    // Skip the first immediate tick — we don't need to refresh on startup.
    interval.tick().await;

    loop {
        tokio::select! {
            _ = shutdown.cancelled() => {
                info!("MTA-STS background refresher shutting down");
                break;
            }
            _ = interval.tick() => {
                refresh_all_policies(&resolver).await;
            }
        }
    }

    info!("MTA-STS background refresher stopped");
}

/// Iterate all cached policies and refresh them.
async fn refresh_all_policies(resolver: &MtaStsResolver) {
    info!("MTA-STS background refresh: starting policy scan");

    let cache = resolver.cache();
    let fetcher = resolver.fetcher();

    // Collect current entries. moka doesn't expose an iterator directly,
    // but we can use `run_pending_tasks` + iterate via the `iter` method
    // available on moka 0.12+.
    let entries: Vec<(String, CachedPolicy)> = cache
        .iter()
        .map(|(k, v)| (k.clone(), v.clone()))
        .collect();

    let total = entries.len();
    let mut refreshed = 0u32;
    let mut failures = 0u32;
    let mut skipped = 0u32;

    for (domain, cached) in entries {
        // Skip mode=none policies (RFC 8461 §8.3 — clean opt-out).
        if cached.policy.mode == PolicyMode::None {
            skipped += 1;
            continue;
        }

        // Look up current TXT record.
        let txt_record = match fetcher.lookup_txt(&domain).await {
            Ok(Some(record)) => record,
            Ok(None) => {
                // TXT record disappeared but we have a cached policy.
                // Keep the cached policy per RFC 8461 §3.3.
                debug!(%domain, "MTA-STS TXT record no longer found during refresh, keeping cached policy");
                skipped += 1;
                continue;
            }
            Err(err) => {
                warn!(%domain, error = %err, "DNS lookup failed during MTA-STS refresh");
                failures += 1;
                continue;
            }
        };

        // If id hasn't changed, just update the fetched_at timestamp to extend freshness.
        if txt_record.id == cached.txt_id {
            let updated = CachedPolicy {
                fetched_at: Instant::now(),
                ..cached
            };
            cache.insert(domain.clone(), updated).await;
            debug!(%domain, "MTA-STS policy unchanged, extended freshness");
            refreshed += 1;
            continue;
        }

        // Id changed — re-fetch the policy.
        debug!(%domain, old_id = %cached.txt_id, new_id = %txt_record.id, "MTA-STS policy id changed, re-fetching");

        match fetcher.fetch_policy(&domain).await {
            Ok(Some(policy)) => {
                let new_cached = CachedPolicy {
                    policy,
                    txt_id: txt_record.id,
                    fetched_at: Instant::now(),
                };
                cache.insert(domain.clone(), new_cached).await;
                info!(%domain, "MTA-STS policy refreshed with new version");
                refreshed += 1;
            }
            Ok(None) => {
                warn!(%domain, "failed to fetch updated MTA-STS policy, keeping cached version");
                failures += 1;
            }
            Err(err) => {
                warn!(%domain, error = %err, "error refreshing MTA-STS policy, keeping cached version");
                failures += 1;
            }
        }
    }

    info!(
        total,
        refreshed,
        failures,
        skipped,
        "MTA-STS background refresh complete"
    );
}
```
